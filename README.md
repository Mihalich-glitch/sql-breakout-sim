## Vulnerability Lab: SQL-injection & XSS Simulator
Учебная лаборатория с открытым исходным кодом, созданная для демонстрации критических уязвимостей веб-приложений. 
Проект позволяет на практике увидеть, как ошибки в фильтрации данных приводят к компрометации базы данных (SQLi) и выполнению произвольного кода на стороне клиента (XSS).

---

## Технологический стек
- Runtime: Node.js (Express)
- Database: PostgreSQL 15 (Alpine)
- Containerization: Docker & Docker Compose
- Frontend: HTML5, CSS3, Vanilla JavaScript


## Сценарии для тестирования
### SQL Injection (Уязвимый поиск)
Используйте поле поиска на главной странице для тестирования следующих векторов атак:

- Обход логики: `1 OR 1=1` — получение всех записей из таблицы пользователей.
- Сбор системных данных: `1 UNION SELECT 1, current_user, 'pass', version()` — вывод версии базы данных и имени текущего пользователя.
- Разведка структуры: `1 UNION SELECT 1, table_name, 'col', 'data' FROM information_schema.tables WHERE table_schema = 'public'` — просмотр списка всех таблиц в базе.

### Как это исправлено? (Defense)
В проекте демонстрируется защита с использованием параметризованных запросов (Prepared Statements). Это разделяет логику SQL-запроса от пользовательских данных.

```
# Уязвимый код (String Concatenation):
const query = `SELECT * FROM users WHERE id = ${userId}`;

# Безопасный код (Parameterized Query):
const query = 'SELECT * FROM users WHERE id = $1';
const result = await pool.query(query, [userId]);
```

### XSS (Cross-Site Scripting)
Демонстрация внедрения и исполнения JavaScript-кода через формы ввода.

Этот модуль демонстрирует одну из самых распространенных уязвимостей фронтенда, при которой вредоносный скрипт сохраняется на сервере и выполняется в браузере каждого посетителя.

#### Реализация уязвимости
1. **Backend (Node.js)**: Принимает данные из формы гостевой книги и сохраняет их в базу данных PostgreSQL без фильтрации HTML-тегов.
2. **Frontend (JavaScript)**: Выводит сообщения из базы на страницу, используя свойство `.innerHTML`. Это заставляет браузер интерпретировать текст из базы как живой HTML/JS код.

#### Визуализация "RAW vs Render"
В проекте реализована уникальная система отладки:
- **Исполняемая область**: Показывает, как браузер отрисовывает код например, выполняет script.
- **Блок "В базе данных (RAW)"**: Показывает «сырой» текст, хранящийся в PostgreSQL. Это наглядно демонстрирует, что база данных лишь хранит данные, а уязвимость возникает в момент их вывода пользователю.

#### Примеры атак (Пресеты):

- **Alert Check**: Базовая проверка доступности скриптов.
- **Cookie Stealing**: Демонстрация того, как скрипт может получить доступ к чувствительным данным сессии.
- **Page Defacement**: Полное изменение внешнего вида страницы (в проекте реализовано через наложение красного полупрозрачного слоя).

### Как это исправлено? (Defense)
В реальных проектах защита реализуется на двух уровнях:

1. **Вывод (Contextual Output Encoding)**: Вместо `.innerHTML` необходимо использовать `.textContent`. Это заставляет браузер отображать теги как обычный текст, не исполняя их.
2. **Очистка (Sanitization)**: Использование библиотек типа *DOMPurify* для удаления опасных тегов вроде *script* или атрибутов *onerror* перед сохранением в базу.

### Инструменты управления
Для удобства демонстрации в проект добавлены:
- Очистка базы: Кнопка быстрого удаления всех XSS-инъекций из БД (DELETE запрос).
- Система подсказок: Динамическое объяснение механики каждой выбранной атаки.
